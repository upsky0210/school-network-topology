# PA 防火牆汰換前檢查表 (v0.7 預備)

本檢查表旨在 PA 防火牆正式汰換前，釐清並降低移轉過程中可能發生的斷網、廣播風暴或服務中斷風險。

## 第一階段：拓樸與實體線路釐清
- [ ] **確認 HPE-5130-01 (192.168.1.23) 實體連線與 VLAN 隔離**
  - **風險**: FortiGate LAN1 (接 5130 Port 48) 與 LAN2 (接 5130 Port 12) 同時進入 5130-01。若 5130 內部未做好 VLAN 隔離，新防火牆上線時極易產生迴圈 (Loop)。
  - **行動**: 必須取得 5130-01 的設定檔或 Read-only 權限，釐清 Port 12, 21, 48 與 Uplink 的 VLAN 狀態。
- [ ] **確認 Cisco 9200 管理資訊與連線狀態**
  - **風險**: 該設備負責全校 FortiAP 無線網路供應，目前 IP、MAC 及接線 Port 皆處於「待確認」狀態。
  - **行動**: 會勘時找出該設備，查明管理 IP，並釐清其 Uplink 是否直接接於原 FortiGate port5，確保新防火牆能正確承接無網段。
- [ ] **釐清 Core01 Port 12 (1F 幼兒園分機櫃)**
  - **風險**: 該埠下掛 101 個 MAC，可能影響學務與行政運作。
  - **行動**: 實地勘查該舊光纖分機櫃，確認是否有非預期的設備串接或環路隱患。

## 第二階段：邏輯網段與服務確認
- [ ] **監視器 (CCTV) 網段隔離評估**
  - **風險**: CCTV 設備目前混入主要 LAN1 網段，汰換過程中可能造成廣播封包過多或資安風險。
  - **行動**: 記錄 CCTV 的 MAC / IP 分佈，評估在新 PA 防火牆上直接切分獨立 VLAN 介面的可行性。
- [ ] **Core01 Port 16 骨幹轉移**
  - **風險**: Port 16 承載 VLAN 1 (Untagged) 及 VLAN 2/3 (Tagged)。任何對 5130-01 或核心的變更都可能導致大範圍斷線。
  - **行動**: 汰換時必須確保新防火牆對接埠的 802.1Q Tagged/Untagged 設定與現有 Core01 邏輯完全一致。

## 第三階段：設備管理與權限
- [ ] **管理權責確認**
  - 確保已發出《廠商唯讀資料請求書》，向維護單位 (中華電信 / BOT 廠商 / 台智光) 取得必要設備的檢視權限。
- [ ] **凍結變更**
  - 在所有「待確認」項目釐清且 PA 汰換完成前，**嚴禁**修改任何現有設備的 SNMP、VLAN 或 Port 狀態。
